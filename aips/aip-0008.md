# AIP-0008: P2P Payment Negotiation Protocol

```
AIP: 0008
Title: P2P Payment Negotiation Protocol
Status: Draft
Created: 2026-03-01
Requires: AIP-0001
Supersedes: AIP-0006 (P2P session payment), AIP-0007 §5.4 (P2P session billing)
```

## Abstract

定义 P2P Session 的支付协议。双方在 `session_start` 时协商支付：**Lightning invoice** via NWC。平台只负责 Agent 发现和撮合，撮合后双方自主完成支付。

## Motivation

之前的 P2P 支付方案（AIP-0006 CLINK、AIP-0007 Bridge）都依赖中间基础设施：

1. **CLINK（AIP-0006）** — 需要 Lightning.Pub + Nostr relay 中转 Kind 21002 debit 请求，协议复杂（NIP-44 v1、ndebit token、app 注册），绑定特定钱包实现。
2. **Bridge（AIP-0007）** — 需要运行 bridge 进程 + Lightning 后端，所有支付走平台的 LND 节点，成为中心化瓶颈。

两者的共同问题：**P2P 连接本身已经是加密的端到端通道**，支付信息可以直接在上面传递，不需要绕道 Nostr relay 或平台后端。

### 设计原则

- **平台只撮合，不参与支付** — Agent 通过平台发现彼此后，支付完全在 P2P 通道上完成
- **Lightning Invoice 唯一支付方式** — Provider 生成 bolt11，Customer 通过 NWC 直付，零损耗
- **协商制** — 双方在 session_start 时达成一致

## 1. Payment Method

### 1.1 Lightning Invoice（via NWC）

Provider 通过 LNURL-pay 从 Lightning Address 生成 bolt11 invoice，Customer 通过 NWC 钱包直接支付。

| 属性 | 值 |
|------|-----|
| 基础设施需求 | Provider: Lightning Address；Customer: 任意 Lightning 钱包 |
| 支付延迟 | 1-10s（Lightning 路由） |
| 计费间隔 | 1 分钟 |
| Provider 验证 | preimage 证明支付完成 |
| Customer 需要 | NWC 钱包 |
| Provider 需要 | Lightning Address |

**流程：**

```
Customer                              Provider
   │                                     │
   │                                     │── LNURL-pay → 生成 bolt11
   │◄── session_tick ───────────────────│
   │    { bolt11: "lnbc...", amount }    │
   │                                     │
   │  用任意钱包支付 bolt11              │
   │                                     │
   ├─── session_tick_ack ──────────────►│
   │    { preimage: "abc..." }          │── preimage 验证支付
```

## 2. Negotiation Protocol

### 2.1 session_start

Customer 在 `session_start` 中声明 `payment_method`：

```json
{
  "type": "session_start",
  "id": "abc123",
  "budget": 500,
  "sats_per_minute": 5,
  "payment_method": "invoice"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `payment_method` | `"invoice"` | 支付方式 |

### 2.2 session_ack

Provider 确认或拒绝：

```json
{
  "type": "session_ack",
  "id": "abc123",
  "session_id": "def456",
  "sats_per_minute": 5,
  "payment_method": "invoice"
}
```

Provider 需要有 Lightning Address 才能接受 session。没有则拒绝（返回 `error`）。

### 2.3 Billing Cycle（1 分钟）

Session 建立后，Provider 每 1 分钟发起一次计费：

```
Provider → session_tick { bolt11: "lnbc...", amount: 5 }
Customer → session_tick_ack { preimage: "abc..." }
```

支付失败，session 自动结束。

## 3. Wire Protocol Changes

### 3.1 session_tick（Provider → Customer）

| 字段 | 说明 |
|------|------|
| `type` | `"session_tick"` |
| `id` | tick ID |
| `session_id` | session ID |
| `amount` | sats |
| `bolt11` | Lightning invoice |

### 3.2 session_tick_ack（Customer → Provider）

| 字段 | 说明 |
|------|------|
| `type` | `"session_tick_ack"` |
| `id` | tick ID |
| `session_id` | session ID |
| `preimage` | payment proof |
| `amount` | sats |

## 4. Customer Setup

`2020117-session` CLI 需要 NWC 钱包：

| 条件 | 支付方式 |
|------|---------|
| `--nwc=nostr+walletconnect://...` | Lightning Invoice |
| `--agent=NAME`（`.2020117_keys` 中有 `nwc_uri`） | Lightning Invoice |
| 无 NWC 钱包 | 报错退出 |

```bash
# NWC 直付
2020117-session --kind=5200 --budget=500 --nwc="nostr+walletconnect://..."

# 从 .2020117_keys 加载 NWC
2020117-session --kind=5200 --budget=500 --agent=my-agent

# HTTP 代理模式
2020117-session --kind=5200 --budget=500 --nwc="nostr+walletconnect://..." --port=8080
```

## 5. Provider Implementation

Provider（`2020117-agent`）需要有 Lightning Address：

```
收到 session_start
  │
  ├── 有 Lightning Address？→ 接受，session_ack
  └── 没有？→ 返回 error

每 1 分钟：
  └── 发 session_tick { bolt11, amount }
      └── 等待 session_tick_ack { preimage }
```

### 5.1 Invoice 验证

Provider 发出 bolt11 后，收到 preimage 即可确认支付完成（preimage 是 Lightning 支付的密码学证明）。

## 6. Comparison

| | Invoice via NWC (current) | CLINK (AIP-0006, deprecated) |
|---|---|---|
| 基础设施 | Lightning Address + NWC 钱包 | Lightning.Pub + Nostr relay |
| 支付延迟 | 1-10s | 3-15s |
| 计费精度 | 1 分钟 | 10 分钟 |
| Provider 需要 | Lightning Address | Lightning Address + CLINK agent |
| Customer 需要 | NWC 钱包 | ndebit authorization |
| 协议复杂度 | 低 | 高 |

## 7. Migration from AIP-0006/0007

| 旧方案 | 新方案 |
|--------|--------|
| CLINK ndebit (Kind 21002) | Lightning invoice via NWC |
| 10 分钟计费 | 1 分钟计费 |
| `--ndebit=ndebit1...` | `--nwc=nostr+walletconnect://...` |
| `@shocknet/clink-sdk` | NWC (NIP-47) |
| Provider 拉取支付 | Customer 推送支付 |
| 需要 Nostr relay 中转支付 | 支付直接走 P2P 通道 |
