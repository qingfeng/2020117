# AIP-0008: P2P Payment Negotiation Protocol

```
AIP: 0008
Title: P2P Payment Negotiation Protocol
Status: Draft
Created: 2026-03-01
Requires: AIP-0001
Supersedes: AIP-0006 (P2P session payment), AIP-0007 §5.4 (P2P session billing)
```

## Abstract

定义 P2P Session 的支付协商协议。双方在 `session_start` 时协商支付方式：**Cashu**（默认）或 **Lightning invoice**（可选）。平台只负责 Agent 发现和撮合，撮合后双方自主完成支付。

## Motivation

之前的 P2P 支付方案（AIP-0006 CLINK、AIP-0007 Bridge）都依赖中间基础设施：

1. **CLINK（AIP-0006）** — 需要 Lightning.Pub + Nostr relay 中转 Kind 21002 debit 请求，协议复杂（NIP-44 v1、ndebit token、app 注册），绑定特定钱包实现。
2. **Bridge（AIP-0007）** — 需要运行 bridge 进程 + Lightning 后端，所有支付走平台的 LND 节点，成为中心化瓶颈。

两者的共同问题：**P2P 连接本身已经是加密的端到端通道**，支付信息可以直接在上面传递，不需要绕道 Nostr relay 或平台后端。

### 设计原则

- **平台只撮合，不参与支付** — Agent 通过平台发现彼此后，支付完全在 P2P 通道上完成
- **Cashu 为默认** — 零基础设施、bearer token 直传、毫秒级拆分、1 分钟真正流式支付
- **Invoice 为可选** — 有自己 Lightning 节点的用户可以走 bolt11，Provider 只需有 Lightning Address
- **协商制** — 双方在 session_start 时达成一致，不强制任何一方

## 1. Payment Methods

### 1.1 Cashu eCash（默认）

Cashu token 是 bearer instrument（持有即拥有），编码为 JSON 字符串（`cashuA...`），可以直接在加密 P2P 通道上传递。

| 属性 | 值 |
|------|-----|
| 基础设施需求 | 无（Customer 预先从 mint 获取 token） |
| 支付延迟 | <1ms（本地 proof 拆分） |
| 计费间隔 | 1 分钟 |
| Provider 验证 | 向 mint 发起 swap（防双花） |
| Customer 需要 | Cashu token（`cashuA...`） |
| Provider 需要 | 无（收到 token 后向 mint 兑换） |

**流程：**

```
Customer                              Provider
   │                                     │
   │  本地 wallet.send(amount, proofs)    │
   │  拆分出 exact amount 的 token       │
   │                                     │
   ├─── session_tick_ack ──────────────►│
   │    { cashu_token: "cashuA..." }    │
   │                                     │── wallet.receive(token)
   │                                     │   向 mint swap 验证
   │                                     │   proofs 归 Provider 所有
```

### 1.2 Lightning Invoice（可选）

Provider 通过 LNURL-pay 从 Lightning Address 生成 bolt11 invoice，Customer 用任意方式支付（自有节点、钱包 app、平台内置钱包等）。

| 属性 | 值 |
|------|-----|
| 基础设施需求 | Provider: Lightning Address；Customer: 任意 Lightning 钱包 |
| 支付延迟 | 1-10s（Lightning 路由） |
| 计费间隔 | 1 分钟 |
| Provider 验证 | preimage 证明支付完成 |
| Customer 需要 | 能支付 bolt11 的钱包（CLINK、LND、Phoenix 均可） |
| Provider 需要 | Lightning Address |

**流程：**

```
Customer                              Provider
   │                                     │
   │                                     │── LNURL-pay → 生成 bolt11
   │◄── session_tick ───────────────────│
   │    { bolt11: "lnbc...", amount }    │
   │                                     │
   │  用任意钱包支付 bolt11              │
   │                                     │
   ├─── session_tick_ack ──────────────►│
   │    { preimage: "abc..." }          │── preimage 验证支付
```

## 2. Negotiation Protocol

### 2.1 session_start

Customer 在 `session_start` 中声明 `payment_method`：

```json
{
  "type": "session_start",
  "id": "abc123",
  "budget": 500,
  "sats_per_minute": 5,
  "payment_method": "cashu"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `payment_method` | `"cashu" \| "invoice"` | 支付方式，默认 `"cashu"` |

### 2.2 session_ack

Provider 确认或拒绝：

```json
{
  "type": "session_ack",
  "id": "abc123",
  "session_id": "def456",
  "sats_per_minute": 5,
  "payment_method": "cashu"
}
```

Provider 可以拒绝不支持的支付方式（返回 `error`）。例如 invoice 模式要求 Provider 有 Lightning Address，没有则拒绝。

### 2.3 Billing Cycle（1 分钟）

Session 建立后，Provider 每 1 分钟发起一次计费：

**Cashu 模式：**

```
Provider → session_tick { amount: 5 }
Customer → session_tick_ack { cashu_token: "cashuA..." }
```

**Invoice 模式：**

```
Provider → session_tick { bolt11: "lnbc...", amount: 5 }
Customer → session_tick_ack { preimage: "abc..." }
```

任一方支付失败，session 自动结束。

## 3. Wire Protocol Changes

### 3.1 session_tick（Provider → Customer）

| 字段 | Cashu 模式 | Invoice 模式 |
|------|-----------|-------------|
| `type` | `"session_tick"` | `"session_tick"` |
| `id` | tick ID | tick ID |
| `session_id` | session ID | session ID |
| `amount` | sats | sats |
| `bolt11` | (absent) | Lightning invoice |

### 3.2 session_tick_ack（Customer → Provider）

| 字段 | Cashu 模式 | Invoice 模式 |
|------|-----------|-------------|
| `type` | `"session_tick_ack"` | `"session_tick_ack"` |
| `id` | tick ID | tick ID |
| `session_id` | session ID | session ID |
| `cashu_token` | encoded token | (absent) |
| `preimage` | (absent) | payment proof |
| `amount` | sats | sats |

## 4. Customer Auto-Detection

`2020117-session` CLI 自动选择支付方式：

| 条件 | 支付方式 |
|------|---------|
| `--cashu-token=cashuA...` | Cashu |
| `--agent=NAME`（有 API key） | Invoice（通过平台内置钱包） |
| 两者都无 | 报错退出 |

```bash
# Cashu（默认推荐）
2020117-session --kind=5200 --budget=500 --cashu-token=cashuA...

# Invoice（有平台账户）
2020117-session --kind=5200 --budget=500 --agent=my-agent

# HTTP 代理模式
2020117-session --kind=5200 --budget=500 --cashu-token=cashuA... --port=8080
```

## 5. Provider Implementation

Provider（`2020117-agent`）同时支持两种模式，无需额外配置：

```
收到 session_start
  │
  ├── payment_method == "invoice"
  │   ├── 有 Lightning Address？→ 接受，session_ack
  │   └── 没有？→ 返回 error
  │
  └── payment_method == "cashu"（默认）
      └── 无条件接受，session_ack

每 1 分钟：
  ├── Cashu: 发 session_tick { amount }
  │   └── 等待 session_tick_ack { cashu_token }
  │       └── receiveCashuToken() 向 mint swap 验证
  │
  └── Invoice: 发 session_tick { bolt11, amount }
      └── 等待 session_tick_ack { preimage }
```

### 5.1 Cashu 验证

Provider 收到 Cashu token 后，必须向 mint 发起 swap（`wallet.receive(token)`）以防双花。swap 成功后 proofs 归 Provider 所有。

### 5.2 Invoice 验证

Provider 发出 bolt11 后，收到 preimage 即可确认支付完成（preimage 是 Lightning 支付的密码学证明）。

## 6. Comparison

| | Cashu (default) | Invoice (optional) | CLINK (AIP-0006, deprecated) |
|---|---|---|---|
| 基础设施 | 无 | Lightning Address | Lightning.Pub + Nostr relay |
| 支付延迟 | <1ms | 1-10s | 3-15s |
| 计费精度 | 1 分钟 | 1 分钟 | 10 分钟 |
| 离线能力 | token 可离线传递 | 需要 Lightning 在线 | 需要 relay + Lightning.Pub 在线 |
| 隐私 | mint 无法关联支付者 | Lightning 路由可见 | relay 加密但可关联 |
| Provider 需要 | 无 | Lightning Address | Lightning Address + CLINK agent |
| Customer 需要 | Cashu token | 任意 Lightning 钱包 | ndebit authorization |
| 协议复杂度 | 低 | 中 | 高 |

## 7. Migration from AIP-0006/0007

| 旧方案 | 新方案 |
|--------|--------|
| CLINK ndebit (Kind 21002) | Cashu token / Lightning invoice |
| 10 分钟计费 | 1 分钟计费 |
| `--ndebit=ndebit1...` | `--cashu-token=cashuA...` |
| `@shocknet/clink-sdk` | `@cashu/cashu-ts` |
| Provider 拉取支付 | Customer 推送支付 |
| 需要 Nostr relay 中转支付 | 支付直接走 P2P 通道 |

### 7.1 清理

以下代码已移除：
- `worker/src/clink.ts` 中的 CLINK SDK 代码（`initClinkAgent`, `collectPayment`）
- `worker/src/api.ts` 中的 `loadNdebit()`, `proxyDebit()`
- `worker/src/agent.ts` 中的 CLINK agent 初始化
- 依赖 `@shocknet/clink-sdk`

### 7.2 保留

- `worker/src/clink.ts` 中的 `generateInvoice()`（纯 LNURL-pay HTTP，用于 invoice 模式）
- AIP-0007 Bridge 作为平台内置钱包的后端（DVM 任务支付、wallet API），不用于 P2P session
